# GrabWP Tenancy v1.1.2

_Released: 2026-06-20 · Previous: v1.1.1_

## Highlights
- Stateless HMAC admin-access tokens replace the on-disk token file — no secret is written to disk anymore.
- Clone completion screen now links straight to the target tenant's editor, homepage, and dashboard.
- Admin UI styling consolidated into a shared stylesheet with utility classes.

## Security
- Replaced the file-based admin access token (a secret written to `tokens.php` and consumed via `rename()`) with stateless HMAC-SHA256 tokens bound to the tenant ID, signed using `AUTH_KEY`/`AUTH_SALT`. Tokens carry a short TTL (default 30 minutes, configurable via the `GRABWP_TENANCY_TOKEN_TTL` constant) and are validated with timing-safe comparisons. No token secret is persisted to disk.

## New
- The clone "complete" screen now shows the target tenant ID along with quick links: Edit tenant, Homepage, and Dashboard.

## Enhancements
- Admin UI refactor: inline styles moved into a new shared stylesheet (`admin/css/grabwp-admin-shared.css`) with reusable utility classes; domain section show/hide now toggles a `hidden` class instead of inline `display`.

## Changes
- Clone pipeline reordered so site URL replacement now runs **before** uploads are copied.
- Installer removes legacy `tokens.php` / `.consumed.*` artifacts on activation; the unused `GrabWP_Tenancy_Path_Manager::get_tokens_file_path()` helper was removed.

## Upgrade notes
- No action required. Existing admin-access URLs from the old token system simply stop working; generate fresh admin-access links from the tenant list after upgrading.
